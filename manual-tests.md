# Ручные тестовые сценарии

Сценарий 1: Удаление статьи из избранного
Описание
Проверка функциональности удаления статьи из избранного и проверки корректного обновления списка избранных статей.

Шаги выполнения

1. Авторизоваться в системе
2. Открыть любую статью
3. Найти кнопку с иконкой сердца (добавить в избранное)
4. Нажать кнопку добавления в избранное
5. Убедиться, что кнопка стала активной (изменила цвет)
6. Нажать на кнопку с иконкой сердца еще раз (удалить из избранного)
7. Перейти в профиль → "Favorited Articles"
8. Проверить, что статья отсутствует в списке избранных

Ожидаемый результат

- Кнопка добавления/удаления работает корректно
- При первом нажатии статья добавляется в избранное
- При втором нажатии статья удаляется из избранного
- Кнопка возвращается в неактивное состояние
- Статья исчезает из списка избранных статей
- Счетчик избранных статей обновляется
- Изменения применяются немедленно

Сценарий 2: Создание статьи с несколькими тегами
Описание
Проверка функциональности создания новой статьи с валидными данными и несколькими тегами, а также проверки корректного отображения тегов.

Шаги выполнения

1. Авторизоваться в системе
2. Нажать кнопку "New Article"
3. Заполнить поле "Article Title" (заголовок: "Тестовая статья с тегами")
4. Заполнить поле "What's this article about?" (описание: "Описание тестовой статьи")
5. Заполнить поле "Write your article" (текст статьи: "Это текст тестовой статьи для проверки работы тегов")
6. Заполнить поле "Enter tags" (несколько тегов через запятую: "playwright, testing, automation, qa")
7. Нажать кнопку "Publish Article"

Ожидаемый результат

- Статья успешно создана
- Отображение заголовка статьи
- Переход на страницу статьи
- Статья появляется в ленте
- Все теги отображаются под заголовком статьи
- Теги являются кликабельными ссылками
- При клике на тег открывается список статей с этим тегом
- Теги корректно форматируются (без лишних пробелов)

Сценарий 3: Добавление множественных комментариев к статье
Описание
Проверка функциональности добавления нескольких комментариев к одной статье и проверки корректного отображения всех комментариев.

Шаги выполнения

1. Авторизоваться в системе
2. Открыть любую статью
3. Прокрутить вниз к секции комментариев
4. Ввести первый комментарий в поле "Write a comment..."
5. Нажать кнопку "Post Comment"
6. Подождать 2-3 секунды
7. Ввести второй комментарий в поле "Write a comment..."
8. Нажать кнопку "Post Comment"
9. Подождать 2-3 секунды
10. Ввести третий комментарий в поле "Write a comment..."
11. Нажать кнопку "Post Comment"

Ожидаемый результат

- Все три комментария добавлены под статьей
- Комментарии отображаются в хронологическом порядке
- Отображение автора "Fusion" для всех комментариев
- Отображение времени добавления для каждого комментария
- Комментарии сохраняются после перезагрузки страницы
- Нет дублирования комментариев
- Поля для ввода очищаются после добавления

Сценарий 4: Смена электронной почты в настройках профиля
Описание
Проверка функциональности изменения email в настройках профиля пользователя и проверки сохранения изменений.

Шаги выполнения

1. Авторизоваться в системе
2. Нажать на имя пользователя "Fusion"
3. Выбрать пункт "Your Settings"
4. Найти поле "Email" (с текущим значением test-20251@mail.ru)
5. Ввести новый email: `new-email@example.com`
6. Оставить остальные поля без изменений:
   - URL of profile picture
   - Username: Fusion
   - Short bio about you
   - Password (оставить пустым)
7. Нажать кнопку "Update Settings"

Ожидаемый результат

- Успешное сохранение изменений
- Отображение уведомления об обновлении
- Новый email сохранен в профиле
- Пользователь остается авторизованным
- Изменения применяются немедленно

Сценарий 5: Поиск и фильтрация статей
Описание
Проверка функциональности поиска и фильтрации контента, включая проверку релевантности результатов.
Шаги выполнения

1. Авторизоваться в системе
2. Найти поле поиска
3. Ввести поисковый запрос
4. Нажать Enter или кнопку поиска
5. Проверить результаты поиска
6. Открыть вкладку "Global Feed"
7. Проверить фильтрацию по тегам

Ожидаемый результат

- Релевантные результаты поиска
- Корректная работа фильтров
- Отображение количества найденных статей
- Быстрый отклик системы
